# Что положить в Memory Bank

## Когда обновляем
- После завершённой задачи (фича, баг, исследование)
- После изменения инфраструктуры/конфигов
- После выявления нетривиального поведения зависимостей

## Что сохраняем
- Назначение и результат (в `.memorybank/product.md` или соответствующем epic/feature)
- Технические детали (duo-файлы, guides, архитектурные заметки)
- Процессы / WF изменения (`.memorybank/wfs/`)
- Ограничения, паттерны, best practices (`docs/architecture/`, `(packages)/`)

## Формат
- Используем существующие шаблоны из [templates/](index.md)
- Обязательно заполняем frontmatter (version, date, status, parent)
- Ставим перекрёстные ссылки (`related_files`, `implementation_files`)

## Проверка себя
- [ ] Информация не дублирует существующие файлы
- [ ] Есть ссылка из индекса раздела
- [ ] Добавлены источники (issue, PR, исследование)
- [ ] Обновлены WF/процессы, если поменялась последовательность шагов
